# TPC-H MonetDB 启动指南

> 当前版本：2026-06-05  
> 示例模型统一使用 `litellm/deepseek/deepseek-v4-flash`。

这份文档只覆盖当前主路径：

```text
TPC-H Q1-Q22
  -> storage plan
  -> base implementation
  -> optimization
  -> MonetDB baseline comparison
  -> outer-loop round decision
```

当前不再把旧时序 workload、旧 HTTP baseline、旧 query-file backend 或旧数据生成器当作可启动路径。启动入口、artifact 命名、prompt 语义和 validator 都以 `tpch_monetdb`、TPC-H、MonetDB 为准。

## 1. 入口与命名

主入口：

```text
run_outer_loop_tpch_monetdb.py
```

调试入口：

```text
python -m tpch_monetdb.run_gen_storage_plan_tpch_monetdb
python -m tpch_monetdb.run_gen_base_impl_tpch_monetdb
python run_optim_loop_tpch_monetdb.py
```

命名规则：

- outer-loop 名称必须以 `outer` 开头，例如 `outer1-2v1`、`outer1-22v1`
- storage plan 子 run 自动派生为 `storageplan...`
- base implementation 子 run 自动派生为 `basef...`
- optimization 子 run 自动派生为 `runoptim...`
- artifact 落盘时会自动加 benchmark 前缀 `tpch_`

例如 `outer1-22v1` 第 1 轮会产生：

```text
storage_plan_runs/tpch_storageplan1-22v1_r001/latest.json
scripted_runs/tpch_basef1-22v1_r001/latest.json
optimization_runs/tpch_runoptim1-22v1_r001/latest.json
outer_loop_runs/outer1-22v1/latest.json
```

## 2. 本机前置

从仓库根目录执行：

```bash
cd "<repo-root>"
```

确认基础工具：

```bash
uv --version
python --version
docker version
rg --version
```

初始化 Python 环境：

```bash
uv sync
uv pip install pymonetdb==1.9.0
```

说明：

- Docker-first 路径会在镜像里安装项目依赖和 `pymonetdb`，host 侧主要负责运行 Docker/Compose
- `uv sync` 和 host 侧 `pymonetdb` 只用于本地 Python 直接运行路径；如果所有测试都在 Docker 里跑，可以不依赖 host Python 连接 MonetDB
- Compose 会把当前仓库 bind mount 到容器内 `/workspace/tpch_monetdb_project`

## 3. LiteLLM / DeepSeek 配置

运行入口会自动读取：

```text
tpch_monetdb/.env
```

第一次先复制模板：

```bash
cp tpch_monetdb/.env_example tpch_monetdb/.env
```

把 `tpch_monetdb/.env` 改成 DeepSeek 原生 LiteLLM provider 配置：

```bash
LITELLM_API_KEY="你的 DeepSeek API Key"

WANDB_ENTITY="learneddb"
WANDB_PROJECT="bespoke-olap-agents"
```

关键点：

- 示例模型是 `litellm/deepseek/deepseek-v4-flash`
- 这个模型走 LiteLLM 的原生 `deepseek/` provider
- 不需要 `LITELLM_BASE_URL`
- 如果 `.env_example` 里保留了 `LITELLM_BASE_URL="https://api.anthropic.com"`，请删除或注释掉
- 只有你明确使用自己的 LiteLLM proxy 时，才设置 `LITELLM_BASE_URL`
- 如果当前 shell 里已经有旧值，先执行 `unset LITELLM_BASE_URL`

当前代码会把 DeepSeek 的默认 reasoning effort 走 DeepSeek chat 参数映射：`thinking.enabled + reasoning_effort=max`。CLI 侧可以显式传 `--reasoning_effort xhigh`，也可以不传，让 DeepSeek 默认值生效。

建议 shell 里固定这两个变量：

```bash
unset LITELLM_BASE_URL
set -a
source tpch_monetdb/.env
set +a

export TPCH_MONETDB_MODEL="litellm/deepseek/deepseek-v4-flash"
export TPCH_MONETDB_REASONING_EFFORT="xhigh"
```

快速确认模型配置解析。Docker-first 路径下在容器工作区里执行：

```bash
python - <<'PY'
from tpch_monetdb.bootstrap_env import bootstrap_runtime_env
from tpch_monetdb.utils.model_setup import setup_model_config

bootstrap_runtime_env()
cfg = setup_model_config("litellm/deepseek/deepseek-v4-flash")
print("use_litellm=", cfg.use_litellm)
print("model_name=", cfg.model_name)
print("accounting_model_name=", cfg.accounting_model_name)
print("base_url=", cfg.base_url)
PY
```

期望：

```text
use_litellm= True
model_name= deepseek/deepseek-v4-flash
accounting_model_name= deepseek-v4-flash
base_url= None
```

如果 `base_url` 打印成 Anthropic 或其它 provider 地址，先修正 `tpch_monetdb/.env`。

## 4. 数据目录

当前 runtime 接受两种 TPC-H `.tbl` 数据目录：

```text
$BASE_DATA_DIR/sf1/customer.tbl
$BASE_DATA_DIR/sf1/lineitem.tbl
...
```

或直接把 8 张表放在根目录：

```text
$BASE_DATA_DIR/customer.tbl
$BASE_DATA_DIR/lineitem.tbl
...
```

本仓库自带 tiny fixture，可用于 smoke 和首次链路验证：

```bash
export BASE_DATA_DIR="$PWD/docker/tpch-monetdb/fixtures/tiny-tpch"
ls "$BASE_DATA_DIR"/*.tbl
```

正式 scale-factor 数据可以放成：

```bash
export BASE_DATA_DIR=/tmp/tpch_monetdb_data
mkdir -p "$BASE_DATA_DIR/sf1"
# 把 TPC-H 8 张 .tbl 表放入 "$BASE_DATA_DIR/sf1/"
```

当前默认 scale factor 是 `sf1`。

## 5. Docker Compose 工作区

Linux 上推荐使用仓库脚本一键构建镜像、启动 MonetDB、导入 tiny TPC-H fixture 并做容器内 smoke check：

```bash
docker/tpch-monetdb/deploy.sh deploy
```

这个 Docker 路径不是“host 连接容器”的瘦 baseline。镜像构建时会复制整个仓库到：

```text
/workspace/tpch_monetdb_project
```

Compose 运行时还会把当前宿主机仓库 bind mount 到同一个目录。也就是说，容器里应该能直接看到：

```text
/workspace/tpch_monetdb_project/tpch_monetdb/misc/tpch
/workspace/tpch_monetdb_project/tpch_monetdb/tests
/workspace/tpch_monetdb_project/run_outer_loop_tpch_monetdb.py
```

常用子命令：

```bash
docker/tpch-monetdb/deploy.sh workspace
docker/tpch-monetdb/deploy.sh test
docker/tpch-monetdb/deploy.sh test tpch_monetdb/tests/test_docker_runtime_harness.py -q
docker/tpch-monetdb/deploy.sh shell
docker/tpch-monetdb/deploy.sh root-shell
docker/tpch-monetdb/deploy.sh monetdb-shell
docker/tpch-monetdb/deploy.sh grant-perf
docker/tpch-monetdb/deploy.sh pmu-smoke
docker/tpch-monetdb/deploy.sh ps
docker/tpch-monetdb/deploy.sh logs
docker/tpch-monetdb/deploy.sh smoke
docker/tpch-monetdb/deploy.sh down
docker/tpch-monetdb/deploy.sh reset
```

建议先检查挂载是否正确：

```bash
docker/tpch-monetdb/deploy.sh workspace
```

成功时会列出 `/workspace/tpch_monetdb_project`，并显示 `tpch_monetdb/misc/tpch` 内容。

容器内跑全部测试：

```bash
docker/tpch-monetdb/deploy.sh test
```

`test` 默认用 root 在容器里执行 pytest，`shell` 和 `root-shell` 也默认以 root 进入容器，避免 Linux bind mount 场景下 `monetdb` 用户没有写临时编译产物或 pytest 临时目录权限。只有需要以数据库服务用户排查时，才用 `monetdb-shell`。

脚本只是 Compose 包装层，不替换容器 entrypoint。可通过环境变量覆盖端口、库名、密码、工作区路径和镜像源：

```bash
TPCH_MONETDB_PORT=50000 \
TPCH_MONETDB_DATABASE=tpch_smoke \
TPCH_MONETDB_PASSWORD=monetdb \
TPCH_MONETDB_WORKSPACE=/workspace/tpch_monetdb_project \
TPCH_MONETDB_WORKSPACE_HOST="$PWD" \
TPCH_MONETDB_PERF_BINARY=/usr/local/bin/tpch-perf \
TPCH_MONETDB_PERF_EVENT_PARANOID=1 \
USE_ALIYUN_DNF_MIRROR=true \
PIP_INDEX_URL=https://mirrors.aliyun.com/pypi/simple \
docker/tpch-monetdb/deploy.sh deploy
```

如果要手动拆开执行，使用下面的 Compose 命令。

先构建 Compose 镜像：

```bash
docker compose -f docker/tpch-monetdb/docker-compose.yml build
```

启动长驻 MonetDB 服务：

```bash
docker compose -f docker/tpch-monetdb/docker-compose.yml up -d tpch-monetdb
```

导入 bundled tiny TPC-H fixture：

```bash
docker compose -f docker/tpch-monetdb/docker-compose.yml \
  --profile init run --rm tpch-monetdb-init
```

成功信号：

- `docker compose ps` 里 `tpch-monetdb` 为 healthy
- `tpch-monetdb-init` 输出 JSON，包含 8 张表的 row count
- 容器内 `deploy.sh smoke` 输出 `smoke_ok lineitem_rows=...`
- 容器内 `/workspace/tpch_monetdb_project/tpch_monetdb/misc/tpch` 可见

当前 Docker 目录是 compose-first：

- 没有仓库自定义 entrypoint
- 没有 `smoke.py`
- Dockerfile 默认使用阿里 PyPI 源安装项目依赖和 `pymonetdb`
- `USE_ALIYUN_DNF_MIRROR=true` 时会尽量把 dnf repo 切到阿里源
- `tpch-monetdb` 和 `tpch-monetdb-init` 都挂载完整仓库到 `/workspace/tpch_monetdb_project`
- `tpch-monetdb` 使用 `privileged: true`、`SYS_ADMIN`、`SYS_PTRACE`、`seccomp=unconfined`、高 `nofile` 和无限 `memlock`
- 默认不添加 `PERFMON` capability；老内核或老 Docker 会因不支持 `CAP_PERFMON` 直接拒绝创建容器
- 镜像内 perf 固定入口是 `/usr/local/bin/tpch-perf`
- `tpch-monetdb` 和 `tpch-monetdb-init` 共享 `/var/monetdb5/tpch-copy`，
  这是 MonetDB `COPY INTO` 读取服务端本地文件所需的临时目录

## 8. 正式 Q1-Q22 outer-loop

确认 Q1-Q2 链路稳定后，再跑 Q1-Q22。先启动 Docker 工作区并进入容器：

```bash
docker/tpch-monetdb/deploy.sh deploy
docker/tpch-monetdb/deploy.sh root-shell
```

下面命令在容器内 `/workspace/tpch_monetdb_project` 执行：

```bash
unset LITELLM_BASE_URL
set -a
source tpch_monetdb/.env
set +a

export TPCH_MONETDB_MODEL="litellm/deepseek/deepseek-v4-flash"
export TPCH_MONETDB_REASONING_EFFORT="xhigh"
export BASE_DATA_DIR="$PWD/docker/tpch-monetdb/fixtures/tiny-tpch"
export ARTIFACTS_DIR="$PWD/tpch_monetdb_artifacts/deepseek_flash_q1_q22_$(date +%Y%m%d_%H%M%S)"

python run_outer_loop_tpch_monetdb.py \
  --conv outer1-22v1 \
  --benchmark tpch \
  --artifacts_dir "$ARTIFACTS_DIR" \
  --base_data_dir "$BASE_DATA_DIR" \
  --validation_mode strict \
  --model "$TPCH_MONETDB_MODEL" \
  --reasoning_effort "$TPCH_MONETDB_REASONING_EFFORT" \
  --auto_u \
  --auto_finish \
  --disable_wandb \
  --disable_tracing
```

默认 outer-loop 参数来自 `tpch_monetdb/config.py`：

- `max_rounds=6`
- `retry_budget=2`
- `stagnant_rounds=3`
- `convergence_threshold=0.02`
- `regression_tolerance=0.05`

如果只想做一次完整 round，可显式加：

```bash
--max_rounds 1
```

## 9. Resume / Retry

同一个 `--conv` 和同一个 `--artifacts_dir` 再跑一次，会从 `outer_loop_runs/<conv>/latest.json` 判断恢复点：

```bash
python run_outer_loop_tpch_monetdb.py \
  --conv outer1-22v1 \
  --benchmark tpch \
  --artifacts_dir "$ARTIFACTS_DIR" \
  --base_data_dir "$BASE_DATA_DIR" \
  --validation_mode strict \
  --model "$TPCH_MONETDB_MODEL" \
  --reasoning_effort "$TPCH_MONETDB_REASONING_EFFORT" \
  --auto_u \
  --auto_finish \
  --disable_wandb \
  --disable_tracing
```

建议：

- 同一个实验重跑时复用相同 `ARTIFACTS_DIR`
- 新实验使用新的 `ARTIFACTS_DIR`
- 不要复用旧 artifact 目录来做不同 query range 的实验

## 13. PMU / perf 可选验收

只有在 Linux 本机 runner 的 Docker 容器里才开启硬件计数器验收。macOS 和 Docker Desktop 不作为最终 PMU 证据。

Linux 上先启动容器并授予 perf 访问：

```bash
docker/tpch-monetdb/deploy.sh deploy
docker/tpch-monetdb/deploy.sh grant-perf
docker/tpch-monetdb/deploy.sh pmu-smoke
```

`grant-perf` 会在 privileged 容器里尝试把 `/proc/sys/kernel/perf_event_paranoid` 设置为 `TPCH_MONETDB_PERF_EVENT_PARANOID`，默认是 `1`。这个值是宿主机内核级设置；不想让脚本改它时，不要运行 `grant-perf`，只运行 `pmu-smoke` 观察当前值。

进入容器：

```bash
docker/tpch-monetdb/deploy.sh root-shell
```

容器内确认：

```bash
/usr/local/bin/tpch-perf --version
cat /proc/sys/kernel/perf_event_paranoid
```

通过后再给 outer-loop 加这些参数：

```bash
--target_cpu native \
--hardware_counter_backend linux_perf_native \
--hardware_counter_runner_cmd /usr/local/bin/tpch-perf \
--host_kernel "$(uname -r)" \
--perf_event_paranoid "$(cat /proc/sys/kernel/perf_event_paranoid)" \
--large_sf 1
```

说明：

- `linux_perf_native` 不是 fallback
- `pmu-smoke` 失败时不要开启 PMU 验收
- tiny fixture 可以验证链路，但不能作为真实性能结论

## 14. 常见故障

### DeepSeek 请求打到错误 endpoint

现象：

```text
model_name= deepseek/deepseek-v4-flash
base_url= https://api.anthropic.com
```

处理：

```bash
# 删除或注释 tpch_monetdb/.env 里的 LITELLM_BASE_URL
unset LITELLM_BASE_URL
set -a
source tpch_monetdb/.env
set +a
```

原生 DeepSeek provider 不需要 `LITELLM_BASE_URL`。

### 缺少 LITELLM_API_KEY

现象：

```text
LITELLM_API_KEY (or provider API key) must be set for litellm models.
```

处理：

```bash
grep -n "LITELLM_API_KEY" tpch_monetdb/.env
unset LITELLM_BASE_URL
set -a
source tpch_monetdb/.env
set +a
```

### 容器内缺少 pymonetdb

现象：

```text
ModuleNotFoundError: No module named 'pymonetdb'
```

处理：

```bash
docker/tpch-monetdb/deploy.sh reset
docker/tpch-monetdb/deploy.sh build
docker/tpch-monetdb/deploy.sh deploy
```

Docker-first 路径不要求 host Python 安装 `pymonetdb`；如果 host 侧直接运行 Python，才需要自行安装。

### MonetDB baseline 连接失败

先确认容器存在：

```bash
docker compose -f docker/tpch-monetdb/docker-compose.yml ps
```

再确认容器内 smoke：

```bash
docker/tpch-monetdb/deploy.sh smoke
docker/tpch-monetdb/deploy.sh workspace
```

如果不通，重建 baseline：

```bash
docker/tpch-monetdb/deploy.sh reset
docker/tpch-monetdb/deploy.sh deploy
docker/tpch-monetdb/deploy.sh workspace
```

如果 init 阶段报类似错误：

```text
when opening /tmp/tpch-monetdb-copy-.../customer.tbl: open failed: No such file or directory
```

说明正在使用旧镜像或旧 Compose 配置。MonetDB 的 `COPY INTO` 在服务容器里读文件，init 容器私有 `/tmp` 对服务容器不可见。更新后应使用共享路径 `/var/monetdb5/tpch-copy`。在 Linux 上执行：

```bash
docker/tpch-monetdb/deploy.sh reset
docker/tpch-monetdb/deploy.sh deploy
```

### artifact 里找不到阶段 summary

确认 `--conv` 和 `--artifacts_dir` 没变：

```bash
find "$ARTIFACTS_DIR" -maxdepth 3 -name latest.json | sort
```

阶段目录应使用 `tpch_` 前缀，例如：

```text
tpch_storageplan1-22v1_r001
tpch_basef1-22v1_r001
tpch_runoptim1-22v1_r001
```

### 想关闭 W&B 和 tracing

启动命令里保留：

```bash
--disable_wandb \
--disable_tracing
```

如果要开启 W&B，去掉这两个参数，并确保 `tpch_monetdb/.env` 里有正确的 W&B 配置。

## 15. 当前推荐顺序

推荐按这个顺序启动：

1. 配置 `tpch_monetdb/.env`，模型使用 `litellm/deepseek/deepseek-v4-flash`
2. `docker/tpch-monetdb/deploy.sh deploy`
3. `docker/tpch-monetdb/deploy.sh workspace`
4. `docker/tpch-monetdb/deploy.sh test tpch_monetdb/tests/test_docker_runtime_harness.py -q`
5. `docker/tpch-monetdb/deploy.sh root-shell`
6. 在容器内跑 `outer1-2v1 --max_rounds 1`
7. 在容器内跑 `outer1-22v1`
8. 检查 `outer_loop_runs/<conv>/latest.json` 和三个阶段 summary
